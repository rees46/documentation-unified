# Список поддерживаемых liquid-переменных

1. `client_id` - идентификатор клиента
2. `first_name` - имя клиента
3. `middle_name` - отчество клиента
4. `last_name` - фамилия клиента
5. `gender` - пол клиента
6. `location` - местоположение клиента
7. `age` - возраст клиента, рассчитанный на основе его даты рождения
8. `birthday` - дата рождения клиента
9. `telegram_id` - идентификатор Telegram клиента
10. `external_id` - внешний идентификатор клиента
11. `loyalty_id` - идентификатор программы лояльности клиента
12. `loyalty_status` - статус клиента в программе лояльности
13. `loyalty_bonuses` - количество бонусов клиента в программе лояльности
14. `loyalty_bonuses_to_next_level` - количество бонусов, необходимых для достижения следующего уровня в программе лояльности
15. `logo_url` - URL логотипа магазина
16. `footer` - футер письма (генерируется методом footer())
17. `phone` - номер телефона клиента
18. `email` - email пользователя или клиента
19. `tracking_url` - URL для отслеживания (для аналитики)
20. `unsubscribe_url` - URL для отписки от рассылки
21. `less_often_url` - URL для изменения частоты получения писем
22. `promocode` - промокод
23. `html_letter` - ссылка на HTML-версию письма (если не тестовый режим)
24. `nps` - индикатор использования NPS и CRM сервисов
25. `tracking_pixel` - пиксель отслеживания для аналитики в email-рассылках
26. `properties` - свойства клиента, если они упоминаются в шаблоне или теме письма и включены соответствующие сервисы (CRM/CDP)
27. `search_groups` - список групп: последние запросы, категории, коллекции Каждая группа имеет свойства: title, rows[name, url, id], remote
28. `search_items` - список найденных товаров
29. `image_url_resized`[SIZE] - список ссылок на отресайзеные изображения Доступны: 120, 140, 160, 180, 200, 220, 310, 520
30. `title` - название для блока рекомендаций
31. `recommended_items` - список рекомендованных товаров
32. `favorite_ids` - массив id товаров через запятую
33. `favorite_items` - массив с избранными товарами. Товары представлены в виде объектов
34. `source_categories` - массив объектов категорий товаров, которые пришли пользователю. Поддерживается в триггерных цепочках
35. `link` - ссылка, может быть использована в шаблонах блока рекомендаций
36. `loyalty_balance_active_bonuses` - текущий активный бонусный баланс
37. `loyalty_balance_pending_bonuses` - бонусы ожидающие активации
38. `loyalty_balance_expiring_bonuses` - cгорающие бонусы (соответствует настройке "Уведомлять о сгорании бонусов за указанное количество дней")
39. `loyalty_actual_level` - текущий уровень программы лояльности
40. `campaign_id` - идентификатор корневой кампании
41. `referral_code` - переменная подставляет в сообщение уникальный идентификатор реферальной программы, заданный по умолчанию
42. `referral_code_ID` - в данном случае вместо ID нужно подставить уникальный идентификатор конкретной реферальной программы, который будет отображаться в сообщении
43. `loyalty_subscription_code` - внутренний код программы лояльности. Информация для маркетолога, с помощью кода можно проверить что за подписка
44. `loyalty_subscription_name` - название подписки программы лояльности
45. `loyalty_subscription_plan_code` - код тарифного плана программы лояльности
46. `loyalty_subscription_plan_name` - название тарифного плана программы лояльности

::: tip Важно

liquid-переменные также используются и в HTTP запросах.

:::
