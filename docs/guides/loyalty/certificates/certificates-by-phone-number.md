# Использование персональных сертификатов

::: tip Коротко о том, как это работает
<details>

В настройках вы разрешаете использовать персональные сертификаты на этапе работы метода `checkout/estimate`. 

То есть, когда платформа будет считать все доступные к применению скидки, бонусы и сертификаты, персональные в этом подсчёте тоже будут учтены. 

При наличии других доступных сертификатов, алгоритм выберет наиболее приоритетный. Не обязательно будет использован именно персональный сертификат.

Если других сертификатов нет, но использование персональных сертификатов не было разрешено - они не будут учитываться в подсчёте оплаты за товары в корзине.

</details>
:::
// TODO иллюстрация и описание по настройкам

Выбор соответствующей настройки в меню активирует флаг `use_personal_certificates`. 

При активации флага, на этапе работы метода `checkout/estimate`, когда рассчитывается сумма к оплате для корзины, к учёту применимых акций, бонусов и сертификатов, добавляются персональные сертификаты, доступные пользователю.

Доступные сертификаты определяются с помощью идентификатора. В рамках программы лояльности это номер телефона. 

Технически процесс выглядит так:

1. Выполняется проверка активности флага `use_personal_certificates` и наличия идентификатора
2. После успешного прохождения, система находит все сертификаты доступные пользователю, включая персональные
3. Сертификаты применяются к корзине

::: tip Для справки

Акции обрабатываются отдельным процессором, поэтому сертификаты, и персональные сертификаты в частности, не влияют на выбор акций. И наоборот.

:::

## Типы сертификатов и ограничения

Метод  `checkout/estimate` работает со всеми сертификатами, доступными пользователю. 

Это подразумевает наличие алгоритмов выбора нужного и сопутствующие ограничения. Три типа сертификатов:

1. Обычный
2. Промо
3. Персональный

::: warning Ключевое ограничение

К корзине может быть применён только один промо-сертификат

:::

Обычные сертификаты могут применяться сразу по несколько штук за раз. Но есть ограничения:

1. Ни при каких условиях нельзя полностью оплатить товар сертификатом полностью. Даже если настройки на стороне магазина предусматривают максимальное долю оплаты сертификатом, покупателю в любом случае нужно будет заплатить хотя бы один рубль
2. Доли оплаты сертификатами устанавливаемые на стороне магазина. Какая часть стоимости товара может быть оплачена сертификатами зависит от заданных параметров в настройках программы лояльности

Промо сертификаты нельзя применить больше одного раза к конкретной корзине. 

Персональные сертификаты работают при успешной идентификации покупателя по номеру телефона.

Номер телефона - главный идентификатор в системе программы лояльности.

## Алгоритм выбора сертификата

Как уже было сказано, метод `checkout/estimate` работает сразу со всеми доступными сертификатами.

Решение какой именно должен быть применён принимается путём многоуровневой сортировки:

1. Доля оплаты с помощью сертификата. Чем меньший процент стоимости товара можно оплатить с помощью сертификата, тем выше его приоритет на применение к корзине
2. Срок годности. Чем ближе сертификат к сроку истечения годности, тем выше его приоритет // TODO - подумать над формулировкой срока годности
3. Дата активации. Чем раньше активирован, тем желательнее его использовать
4. Остаток на балансе. Чем меньше оставшаяся сумма, тем вероятнее будет использован сертификат
5. ID (сортировка по возрастанию)

Таким образом, логика алгоритма заключается в сведении к минимуму сгорания неиспользованных сертификатов и оставшихся на них средств. 

## Жизненный цикл сертификата

| Этап          | Метод               | Действие                                                                              | Статус сертификата                                                 |
|---------------|---------------------|---------------------------------------------------------------------------------------|--------------------------------------------------------------------|
| Расчет        | `checkout/estimate` | Проверка, какие сертификаты можно применить                                           | `active` (не меняется)                                             |
| Применение    | `checkout/apply`    | Создание транзакций списания                                                          | `active` → `finished` (баланс меняется)                            |
| Подтверждение | `checkout/confirm`  | Подтверждение (статус сертификата не меняется)                                        | `finished` (не меняется)                                           |
| Отмена        | `checkout/cancel`   | Возврат баланса, создание возвратных транзакций                                       | `finished` → `active` (баланс восстанавливается)                   |
| Изменение     | `checkout/change`   | Изменение суммы списания ( в связи с изменением состава/количества товаров в корзине) | `finished` → `active` (частично, если баланс не обнулен полностью) |
