# MCP сервер REES46

MCP сервер REES46 - это надстройка над вашей существующей СDP/маркетинг платформой REES46, которая позволяет использовать LLM модели как «AI  аналитика» на основе  вашей клиентской базы и реальных маркетинговых данных

MCP (Model Context Protocol) — стандарт, через который LLM подключается к вашим данным и инструментам.

REES46 остаётся системой хранения и обработки данных (CDP, поведение, заказы, кампании), MCP сервер делает эти данные доступными и понятными модели.

LLM отвечает на вопросы, пишет отчёты, строит инсайты и рекомендации на основе реальных данных вашей платформы, а не абстрактных знаний.

На базе MCP сервера вы можете строить собственные LLM сценарии под любые задачи: от маркетинговой аналитики и CRM отчётов до узкоспециализированных ассистентов для конкретных ролей и процессов. 

## Функциональные возможности (в связке REES46 + LLM)

 1. Доступ к данным без SQL и BI
Через AI аналитика можно на естественном языке:
-	Спрашивать: «Покажи динамику выручки и заказов по месяцам за последний год», «Сколько активных клиентов за 90 дней?»
-	Запрашивать выборки: топ сегменты по выручке, клиенты с ростом/падением покупок, отклик на маркетинговые кампании
LLM работает с данными REES46 не напрямую, а через MCP-сервер, который обеспечивает безопасный и стандартизированный доступ:
-	к базам данных 
-	пользовательскому поведению 
-	сегментам 
-	аналитическим метрикам
2. Аналитические отчёты
Продукт может формировать разные кросс функциональные типы отчетов, например:
-	анализ продаж, выручки и конверсии в разных срезах
-	отчеты по клиентам, сегментации и поведению
-	анализ эффективности маркетинговых кампании и каналов
-	анализ массовой и триггерной коммуникации: доставки, открытия, клики, конверсии, отписки
-	отчеты по удержанию, повторным покупкам и LTV
-	анализ продуктовых метрик и использованию функций
-	сводные отчеты по ключевым KPI за период, и т.д.
На выходе вы получаете:
-	текстовые выводы и инсайты
-	графики, таблицы и структуры, пригодные для экспорта в Excel/BI
-	краткое резюме для руководства
3. Генерация рекомендаций по CRM и маркетингу
На основе данных «AI аналитик» может:
-	предлагать, какие сегменты нужно удерживать, реанимировать, масштабировать
-	давать идеи кампаний: кому предложить персональные скидки, кросс селл, реактивацию
-	помогать в постановке A/B тестов: формулировать гипотезы и метрики успеха, выбирать сегменты
-	помогать переформатировать триггерные сценарии и массовую коммуникацию
::: tip Важно
Рекомендации строятся на фактах из ваших данных, но подаются как гипотезы и потенциальные действия, а не как автоматические решения.
:::     
4. Быстрые ответы на тактические вопросы бизнеса
Команды могут задавать «человеческие» вопросы:
-	«Какие сегменты принесли больше всего выручки в апреле?»
-	«Кто у нас на грани ухода среди клиентов с высоким LTV?»
-	«Какие кампании хуже всего отработали для сегмента AX?»
«AI аналитик» переводит это в запросы к REES46 через MCP и возвращает структурированный ответ.
5. Разнообразие: собственные AI-сценарии под любые задачи 
Важно: на базе MCP‑сервера, в отличии от других AI-продуктов, вы можете строить любые собственные AI‑сценарии под ваши задачи: от маркетинговой аналитики и CRM‑отчётов до узкоспециализированных ассистентов для конкретных ролей, команд и процессов.
6. Ограничения и рамки ответственности
AI‑аналитик — инструмент для ускорения аналитики и генерации гипотез, а не полная замена аналитики:
-	Важные решения (изменение цен, значимые бюджеты, кредитные/рисковые решения) должны проверяться отдельными инструментами и экспертами
-	Продукт - инструмент для ускорения аналитики и генерации гипотез, а не полная замена аналитики
-	AI‑аналитик не изменяет данные, сегменты, кампании и настройки REES46 — он только читает данные, строит выводы и предлагает действия
-	Все исходные данные берутся из REES46/API, а метрики и выводы рассчитываются и интерпретируются LLM на их основе
-	Продукт предназначен для маркетинга, CRM и продуктовой аналитики, а не для фин. учёта, юридически значимой отчётности или автоматического принятия риск‑решений
-	Важные решения (изменение цен, значимые бюджеты, кредитные/рисковые решения) должны проверяться отдельными инструментами и экспертами
