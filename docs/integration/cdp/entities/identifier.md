# Идентификатор клиента

Самая главная задача CDP — в любой момент в режиме реального времени определять, кто именно пользуется устройством. Для передачи устройства между людьми используются их идентификаторы.

CDP работает с разными идентификаторами клиентов:

- `email`;
- `phone`;
- `loyalty_id` — идентификатор клиента во внешней программе лояльности;
- `telegram_id` — идентификатор клиента из Telegram-бота;
- `external_id` — сквозной идентификатор клиента в вашей базе данных.

Простейший вариант, когда в интеграции используется один идентификатор, который является логином клиента в вашей системе. Например, `email` или `phone`. При получении идентификатора вместе с `did` CDP переключает устройство на владельца этого идентификатора. 

Сложнее, когда используются одновременно два идентификатора. Еще сложнее, когда данные конфликтуют и в запросе приходят `email` одного клиента и `phone` другого. О нашем способе решения задачи [можно прочитать в статье](https://rees46.ru/blog/omnichannel-user-matching/).
