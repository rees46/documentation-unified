# CDP

Customer Data Platform – платформа пользовательских данных, позволяющая в режиме реального времени дедуплицировать посетителей, в том числе анонимных, и аккуратно собирать и обрабатывать поведенческие данные каждого пользователя или устройства.

В процессе интеграции вы настраиваете:

1. Передачу событий (действий) пользователей, включая анонимных.
2. Передачу свойств пользователей.
3. Передачу заказов и их статусов.

## Основные сущности CDP

### Device ID

Технически мы отличаем устройство от человека. Три главных принципа:

1. У одного человека может быть несколько устройств: смартфон, ноутбук, планшет, компьютер. Данные со всех устройств одного человека должны быть собраны в единый профиль клиента.
2. Одно устройство в любой момент времени может принадлежать только одному человеку.
3. Устройство может сменить владельца: например, домашний компьютер, на котором по очереди могут работать муж, жена и ребенок. Данные, полученные при смене владельца, должны быть собраны в профиль того человека, который пользуется этим устройством на данный момент.

Поэтому SDK при старте сессии и трекинге действий передают данные не по пользователю, а по устройству. За это отвечает параметр `did` (device ID). SDK автоматически создает `did` для каждого нового посетителя и использует его для привязки данных к событиям.

:::info Интеграция S2S
При интеграции [server-to-server](../extended/s2s.md) вам придется самостоятельно генерировать `did`, блокировать поисковых роботов и других ботов.
:::

### Сессия

Сессия позволяет отделять разные сеансы посещения сайта посетителем (на самом деле - устройством). Началом сессии считается первое действите посетителя после минимум двухчасового отсутствия событий.

Для маркировки сессии используются параметры `sid` (актуальный) и `seance` (устаревший, оставлен для обратной совместимости и со временем будет убран).

### Стрим

Тег источника данных. Этот параметр устанавливается в [инициализации SDK](./sdk.md). Если не указан, считается, что поток данных идет с сайта.

Можно задавать разные стримы для точек касания с клиентом: сайт, мобильные приложения, кассы, инфокиоски, чат-боты, колл-центр и пр. 

Формат значения: строка латиницей. Мы рекомендуем использовать простые короткие значения типа `ios`, `android`, `pos`. Если используете только сайт, можно не задавать значение `stream`, тогда платформа сама будет считать, что эти данные пришли с веб-сайта.
