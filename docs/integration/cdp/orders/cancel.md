# Отмена заказов

Метод используется для полной отмены заказа. Для частичного возврата используйте [метод синхронизации заказов](./sync.md).

Эндпоинт отмены заказа такой же, как в синхронизации. Разница лишь в том, что в списке заказов достаточно передавать идентификатор заказа и статус отмены.

Рекомендуется отправлять заказы партиями не более 5000 штук в одном запросе.

В случае невалидных данных будет отправлено уведомление на email, отмеченный для технических уведомлений в списке сотрудников в личном кабинете.

```
POST https://api.rees46.ru/sync/orders
```

## Параметры

### Структура тела запроса

| Параметр     | Обязателен? | Описание                |
|--------------|-------------|-------------------------|
| shop_id      | Да          | API-ключ                |
| shop_secret  | Да          | Секретный API-ключ      |
| orders       | Да          | Массив заказов          |

### Структура объекта заказа

| Параметр                   | Обязателен? | Описание                                                                                                                                                   |
|----------------------------|-------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| id                         | Да          | Номер заказа                                                                                                                                               |
| status                     | Да          | Статус заказа из таблицы соответствия                                                                                                                      |

## Запрос

Запрос:

```shell 
curl -i -X POST -H "Content-Type: application/json" --data-binary "@data.json" https://api.rees46.ru/sync/orders
```

Тело запроса:

```json data.json
{
  "shop_id": "DvLWN2ZTMZ",
  "shop_secret": "EIxTuot8sj",
  "orders": [
    {
      "id": "yKsvZbWpCL",
      "status": "Cancelled"
    },
    {
      "id": "aNszQbWPuN",
      "status": "Refunded"
    }
  ]
}
```


## Ответ

Пример ответа сервера:

```json 
{
  "status": "success"
}
```

В случае ошибки ответ будет в формате:

```json 
{
  "status": "error",
  "payload": {
    "message": "Shop not found"
  }
}
```

:::info Важно
API не валидирует данные заказов синхронно. Метод принимает объект с заказами и отправляет в очередь для обработки. В ответ отдает `200 OK`.
Если в запросе есть невалидные заказы, то уведомление о них придет по электронной почте сотрудникам, которые подписаны на технические уведомления в личном кабинете.
:::
