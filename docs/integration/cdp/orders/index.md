# Работа с заказами и их статусами

При [трекинге заказа](../standard-events/purchase.md) с фронта в CDP создается заказ с переданными в нем товарами. 
Но это только начальный этап жизненного цикла заказа.
API для работы с заказами служит для решения нескольких задач:

1. Первичный импорт истории заказов.
2. Регулярное обновление статусов и состава заказов клиентов.
3. Импорт оффлайн-заказов из розничных точек.

## Заказы CDP и заказы программы лояльности
Важно учитывать, что заказы, импортируемые в CDP, и заказы программы лояльности – это разные сущности. 

Заказы ПЛ строго привязаны к идентификатору участника ПЛ, и привязка не может быть изменена.

Заказы CDP привязаны к [первичному идентификатору CDP](../entities/identifier.md) и могут в редких случаях меняться, если вы выполняете корректировку профиля клиента (например, объединение профилей).
