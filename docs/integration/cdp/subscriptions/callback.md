# Сервисные операции с email

Метод позволяет отмечать email системными флагами:

- жалоба на спам;
- hard bounce;
- отписка;
- попадание в черный список.

Как правило, метод используется в виде коллбэка на внешние события и нужен только, если у вас используется свой собственный канал отправки сообщений и получения реакции на них.

```
POST https://api.rees46.ru/subscriptions/callback
```

## Параметры

| Параметр                      | Обязателен? | Описание                                 |
|-------------------------------|-------------|------------------------------------------|
| shop_id                       | Да          | API-ключ                                 |
| shop_secret                   | Да          | Секретный API-ключ                       |
| email                         | Да          | Email клиента                            |
| event                         | Да          | Событие, произошедшее с email (см. ниже) |

## Виды событий

| Значение параметра `event` | Интерпретация                                                    |
|----------------------------|------------------------------------------------------------------|
| `complained`               | Пришла жалоба клиента на спам (по телефону, тикет или через FBL) |
| `hard_bounced`             | Попытка отправки завершилась ошибкой, что адрес не существует    |
| `blacklisted`              | Email добавлен в черный список                                   |
| `unsubscribed`             | Email отписался от рассылок                                      |


## Запрос

Пример запроса:

```shell [S2S]
# Полный пример
curl 'https://api.rees46.ru/subscriptions/callback' \
    -X 'POST' \
    -H 'Content-Type: application/json' \
    --data-raw '{"shop_id": "...", "shop_secret": "...", "email": "...", "event": "..."}'

```

## Ответ

Пример ответа сервера:

```json 
{
  "status": "success"
}
```

В случае ошибки ответ будет в формате:

```json 
{
  "status": "error",
  "payload": {
    "message": "Invalid secret key"
  }
}
```
