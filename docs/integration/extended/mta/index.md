# Настройка и работа с почтового кластера

Для того, чтобы почтовые серверы (MTA) не были заблокированы в почтовых сервисах (Mail.ru, Yandex, Google и так далее), необходимо:

1. Вести постоянный мониторинг репутации IP-адресов и доменов отправителя в почтовых сервисах.
2. Обрабатывать жалобы на спам (FBL).
3. Обрабатывать баунсы (hard bounce, soft bounce): отчеты почтовых сервисов о невозможности доставки сообщений.

Данный раздел актуален для кластеров on-premise и white-label. В SaaS-кластере все реализовано и обслуживается командой REES46.
