# Домены для отправки, SPF, DKIM, DMARC

Для каждого типа email рассылки рекомендуется использовать отдельный поддомен отправителя. Таким образом для массовых, триггерных и транзакционных рассылок будет отдельный домен со своей репутацией. И, если даже по каким-то причинам домен для массовых рассылок попадет в бан, ваши транзакционные рассылки со статусами заказов и восстановлением пароля будут доставляться без проблем.

Примеры доменов:

- `b.e.mydomain.com` — для массовых рассылок
- `t.e.mydomain.com` — для триггерных рассылок
- `m.e.mydomain.com` — для транзакционных рассылок

Эти домены затем прописываются в личном кабинете для соответствующих типов рассылок.
