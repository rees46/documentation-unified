# Начисление бонусов за действие

Позволяет начислять бонусы за действия. Например: установка приложения, регистрация в программе лояльности (приветственный бонус), заполнение анкеты, день рождения и т.д.

Метод требует указания параметра `event`, за который выполняется вознаграждение. Для одного и того же `event` бонусы можно начислить только один раз. Это гарантирует, что покупатель не сможет получить вознаграждение дважды.

Бонусы за действия можно [отменять по API](./cancel.md).

:::info Расширенная CDP
Того же результата можно добиться, начисляя бонусы через кастомное событие расширенной CDP. Управление происходит в личном кабинете, поэтому после первичной интеграции ресурсы разработчика не потребуются.
:::

:::warning Устарело
Данный метод со временем будет перенесён на другой endpoint. По возможности используйте начисление за кастомное событие.
:::


```
POST https://api.rees46.ru/loyalty/basic/event_bonus
```

## Параметры

| Параметр    | Обязателен? | Описание                                                                                                                                                         |
|-------------|-------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| shop_id     | Да          | API-ключ                                                                                                                                                         |
| shop_secret | Да          | Секретный ключ API                                                                                                                                               |
| identifier  | Да          | Номер телефона клиента, идентификатор участника ПЛ                                                                                                               |
| event       | Да          | Имя события, за которое начисляем бонусы. Английские буквы и цифры                                                                                               |
| amount      | Да          | Сумма начисления                                                                                                                                                 |
| expire_in   | Нет         | Дата сгорания начисленных бонусов в формате `YYYY-MM-DD`. Если не указана, будет взято значение по-умолчанию из настроек программы лояльности или уровня клиента |

## Запрос

Пример запроса:

```shell 
curl --header "Content-Type: application/json" \
  --request POST \
  --data-binary "@payload.json" \
  https://api.rees46.ru/loyalty/basic/event_bonus
```

Пример JSON-тела:

```json payload.json
{
    "shop_id": "...",
    "shop_secret": "...",
    "identifier": "7XXXXXXXXX",
    "event": "registration",
    "amount": 5000,
    "expire_in": "2030-01-01"
}
```


## Ответ

Пример ответа сервера:

```json 
{
  "success": true,
  "transaction_id": 3373
}
```


В случае ошибки ответ будет в формате:

```json 
{
  "success": false,
  "payload": {
    "message": "Identifier is not valid"
  }
}
```
