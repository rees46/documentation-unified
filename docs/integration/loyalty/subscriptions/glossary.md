# Термины и сущности

## Подписка

Платное или бесплатное членство, представленное компанией, которая через свои планы и бенефиты конфигурирует правила начисления/списания бонусов, уровни скидок, бесплатные услуги и прочие привилегии, пока подписка находится в активном состоянии.

В одной группе компаний может быть несколько подписок. Например:

1. Подписка на маркетплейсе
2. Подписка в стриминговом сервисе

У подписки могут быть следующие статусы:

1. Неактивна — по-умолчанию
2. Активна
3. В архиве

Заархивированные подписки не выводятся в списке подписок в API и недоступны клиентам. Перед архивированием подписки необходимо перевести клиентов на другую подписку или отключить их.

## План (тариф)

Аналог "уровня" в программе лояльности. План представляет собой уровень привилегий пользователя с определенными лимитами для этого уровня в рамках подписки. 

Если подписка не подразумевает нескольких уровней, достаточно создать только базовый тарифный план.

У плана могут быть следующие статусы:

1. Неактивен — по-умолчанию
2. Активен
3. В архиве

Заархивированные планы не выводятся в списке подписок в API и недоступны клиентам. Перед архивированием плана необходимо перевести клиентов на другой активный план.

## Бенефит

Конкретная "плюшка" для клиента. Может быть разных типов:

1. Булевый: либо "есть" либо "нет". Например: доступ к определенной функции сервиса. 
2. Числовой: задает максимальное количество раз получения бенефита в расчетный период. Например: 3 бесплатных доставки в месяц.
3. Акции: дает доступ к определенным акциям с возможностью указать максимальный накопленный лимит скидок по этим акциям в расчетный период. Например: "Скидка 5%" на "Базовом" тарифном плане и "Скидка 10%" на "Расширенном" тарифном плане. 

У бенефита могут быть следующие статусы:

1. Неактивен — по-умолчанию
2. Активен
3. В архиве

Заархивированные бенефиты недоступны клиентам и не участвуют в расчете скидок и вознаграждений.


## Клиент

Конечный пользователь сервиса или покупатель в магазине.
