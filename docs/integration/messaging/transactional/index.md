# Транзакционные рассылки

Транзакционные рассылки — это сервисные сообщения, которые могут ставить клиента в известность, что его заказ ушел на склад или приехал в пункт выдачи. У них нет задачи продать, только информировать, в отличие от маркетинговых рассылок.

:::info Репутация email-домена
Для транзакционных рассылок необходимо иметь отдельный почтовый домен с чистой репутацией. Если отправлять такие рассылки от общего рекламного домена, такие рассылки могут попадать в спам.
:::

Частые кейсы для транзакционных рассылок:

1. Изменение статуса заказа.
2. Восстановление пароля.
3. Одноразовые коды.
4. Уведомление об изменении бонусного баланса.

Транзакционные рассылки отправляются не только по email. Может быть задействован любой канал. Например, одноразовые коды для 2FA-авторизации чаще отправляются по SMS.

## Создание клиента

Если пользователь отсутствует в CDP, отправка сообщения создаст этого пользователя с теми контактными данными, которые пришли в запросе:

- email;
- phone;
- external_id;
- loyalty_id;
- telegram_id.
