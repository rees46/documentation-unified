# Список запросов с пустыми результатами

Запросы, по которым ничего не находится, пишутся в базу данных и доступны для анализа как в личном кабинете, так и по API.

```
GET https://api.rees46.ru/search/no_result_queries
```

## Параметры

| Параметр    | Тип     | Обязателен? | Описание           |
|-------------|---------|-------------|--------------------|
| shop_id     | String  | Да          | API-ключ           |
| shop_secret | String  | Да          | Секретный API-ключ |

## Запрос

Пример запроса:

```shell [S2S]
curl https://api.rees46.ru/search/no_result_queries?shop_id=...&shop_secret=...
```

## Ответ

Пример ответа сервера:

```json
[
  {
    "query": "iphone 4s",
    "quantity": 3844
  },
  {
    "query": "covfefe",
    "quantity": 384491
  }
]
```

:::warning Внимание

В будущем структура будет приведена к стандарту ответа:

```json
{
  "status": "success",
  "payload": {
    "no_result_queries": [
      {
        "query": "iphone 4s",
        "quantity": 3844
      },
      {
        "query": "covfefe",
        "quantity": 384491
      }
    ]
  }
}
```

Рекомендуем подготовить обработчик ответа под новую структуру, чтобы ваша интеграция не сломалась.
:::


Расшифровка свойств ответа:

| Свойство | Тип      | Описание                                                                         |
|----------|----------|----------------------------------------------------------------------------------|
| query    | String   | Поисковый запрос                                                                 |
| quantity | Integer  | Сколько раз этот поисковый запрос был запрошен клиентами за последние две недели |

В случае ошибки ответ будет в формате:

```json 
{
  "status": "error",
  "payload": {
    "message": "Shop not found"
  }
}
```
