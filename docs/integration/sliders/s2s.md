# S2S запросы к слайдерам

Если вы хотите получить информацию о слайдере на бэкенде и затем самостоятельно отрисовать их на фронте, вы можете использовать данный метод:

```
GET https://api.rees46.ru/slider
```

## Параметры

| Параметр     | Обязателен? | Описание                                                                                                    |
|--------------|-------------|-------------------------------------------------------------------------------------------------------------|
| shop_id      | Да          | API-ключ                                                                                                    |
| code         | Да          | Уникальный код слайдера                                                                                     |
| did*         | Да          | [Идентификатор устройства](../entities/did.md)                                                              |
| email*       | Да          | Email клиента                                                                                               |
| phone*       | Да          | Телефон клиента                                                                                             |
| external_id* | Да          | Внешний идентификатор клиента                                                                               |
| tags         | Нет         | Теги для фильтрации баннеров. Несколько тегов указываются через запятую. Теги заполняются в личном кабинете |

:::info Идентификаторы
Параметры, отмеченные [*] — обязателен хотя бы один из них. На стороне сайта или мобильных приложений SDK автоматически передает `did`.
:::

## Запрос

```shell [S2S]
curl 'https://api.rees46.ru/slider?shop_id=...&code=...&did=...&email=...&phone=...&external_id=...&tags=...'
```

## Ответ

Пример ответа сервера:

```json 
{
  "status": "success",
  "payload": {
    "properties": {
      "code": "homepage_mobile",
      "width": 1000,
      "height": 160
    },
    "slides": [
      {
        "image_url": "...",
        "url": "...",
        "deeplink_ios": "...",
        "deeplink_android": "...",
        "content_type": "...",
        "content": "...",
        "html": "..."
      },
      ...
    ]
  }
}
```

Расшифровка свойств ответа:


| Property                  | Type    | Description                   |
|---------------------------|---------|-------------------------------|
| properties                | Object  | Общие свойства слайдера       |
| properties.code           | String  | Уникальный код слайдера       |
| properties.width          | Integer | Ширина блока в пикселях       |
| properties.height         | Integer | Высота блока в пикселях       |
| slides[]                  | Array   | Массив баннеров в слайдере    |
| slides[].image_url        | String  | Ссылка на изображение         |
| slides[].url              | String  | Ссылка на посадочную страницу |
| slides[].deeplink_ios     | String  | Диплинк для iOS               |
| slides[].deeplink_android | String  | Диплинк для Android           |
| slides[].content_type     | String  | Тип контента (служебное поле) |
| slides[].content          | String  | Контент (служебное поле)      |
| slides[].html             | HTML    | Готовая верстка слайдера      |

В случае ошибки ответ будет в формате:

```json 
{
  "status": "error",
  "payload": {
    "message": "Invalid slider code"
  }
}
```
